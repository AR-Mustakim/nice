Adaptation Problem Domain Ontology


NiSE Framework for Self-Adaptive System

Period: 2012.07. - 2017.06.

Description: This project mainly aims to develop ontological engineering process for self-adaptive software intelligence to analyze problems, if the problem occurs by changes of a situation and environment. For the self-adaptation to the various known and unknown situation, the system should have proper knowledge and the relationship between them. Knowledge about the system is classified with the different abstraction levels from goal model to role-based architecture model. Between two models, the feature model is defined to connect between abstract layer to actual architecture layer. In addition to knowledge about the system itself, there are environmental models such as context model, policy model and engineering process model. These various types of knowledge and relationship are defined as the ontology. During the adaptation process, the system asks many questions to deal with the various situations and problems. The answer is derived from the diverse types of knowledge and the relationships between them. Making the appropriate questions and deriving answer from knowledge and relationship is very important.
